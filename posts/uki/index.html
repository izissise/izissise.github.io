Unified Kernel Image

A Unified Kernel Image (UKI) is a combination of an UEFI boot stub program, a Linux kernel image, an initrd, and further resources in a single UEFI PE file. This file can either be directly invoked by the UEFI firmware or through a boot loader.

UKIs can run on UEFI systems and simplifies the distribution of small kernel images, for example it simplify network booting with IPxe.

UKIs make rootfs and kernels composable, making possible to derive a rootfs to multiple kernel version with one file for each pairs.

The simple way

If you use a systemd based OS and you want to use UKIs, you're probably better off using systemd-boot.

In this post with want to create a ready to boot UKI for another machine than the one we creating it from.

Linux EFI

Linux kernel should be compiled with CONFIG_EFI_STUB=y see https://docs.kernel.org/admin-guide/efi-stub.html

Hint

When building the kernel with EFI support, the final artifact name in the arch/"$ARCH"/boot/ directory changes for X86_64 it will be name bzImage but for other archs it will be vmlinuz.efi

Lets us find pre-compiled kernel on archlinux repository[1]:

export klatest=$(curl 'https://archive.archlinux.org/packages/l/linux/' | xmllint --recover --html --xpath 'string(//a[last() - 1]/@href)' -)
# We're only interested in the vmlinuz file
curl "https://archive.archlinux.org/packages/l/linux/${klatest}" | tar --zstd --wildcards --no-anchored  --transform='s:.*/::' -xf - 'vmlinuz'

With this you can already boot linux directly from the UEFI

qemu-system-x86_64 -m 512M -enable-kvm -nographic -cpu host -bios /usr/share/ovmf/OVMF.fd -drive file=fat:rw:./,format=raw
# Press ESC to enter uefi shell
fs0:vmlinuz console=ttyS0

The kernel crashed because there is no root FS but it successfully booted as an EFI app.

SD Boot

UEFI doesn't support UKIs directly hence we need glue code to copy UKI sections to memory and pass control to Linux.

We need to compile systemd-boot as the bootloader, to put in the UKI so it copies linux and initrd at the right place, display the splash screen measure secureboot values and finally give control to linux with the cmdline and dtb.

git clone https://github.com/systemd/systemd.git
cd systemd
meson setup build
cd build
ninja src/boot/efi/linuxx64.efi.stub
cp src/boot/efi/linuxx64.efi.stub ../../uki_base.efi

Populating UKI

UKIs is PE file layout with standarized sections:

  • .linux where we put the kernel we can just copy vmlinuz
  • .osrel a key value file with metadata about the booted image
  • .initrd the initramfs that will be loaded with kernel
  • .splash an image displayed at boot
  • .dtb a description of devices on the host machine
  • .sbat security
  • .pcrsig a signature for secure boot
  • .pcrpkey the key for secure boot

Previously we build a minimal rootfs that we will be able to use in our UKI

Creating an UKI involves mainly objcopy with the right parameter to pack all data into one PE file

we have uki_base.efi our base UKI image, we now just have to fill the aforementioned sections.

the composition can be done in any order using multiple call to objcopy

# cmdline.txt
console=ttyS0

# osrel.txt
NAME=UKI
PRETTY_NAME=UKI
VERSION_CODENAME=1
ID=1
BUILD_ID=1

sd_boot randomize the virtual memory address (vma) so we first need to get the IMAGE_BASE to put the section after the sd_boot. we give enough room between sections vma so they don't overlap for big files

IMAGE_BASE=$(objdump -x "uki_tmp.efi" | grep ImageBase | awk '{print $2}')
objcopy \
    --add-section .osrel=osrel.txt \
    --add-section .cmdline=cmdline.txt \
    --add-section .initrd=initramfs.cpio \
    --add-section .linux=vmlinuz \
    --change-section-vma .osrel=$( (( off = 16#00020000 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --change-section-vma .cmdline=$( (( off = 16#00020200 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --change-section-vma .initrd=$( (( off = 16#00100000 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --change-section-vma .linux=$( (( off = 16#02000000 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --set-section-flags .linux=code,readonly \
    uki_base.efi bootx64.efi

Test with Qemu

Create a directories and file efi/boot/bootx64.efi[2], then run Qemu:

mkdir -p efi/boot
cp bootx64.efi efi/boot/
qemu-system-x86_64 -m 512M -enable-kvm -nographic -bios /usr/share/ovmf/OVMF.fd -cpu host -drive file=fat:rw:./,format=raw

Conclusion

We created a single UKI file that contains a kernel, initramfs, cmdline and metadata. It is easy to distribute and boot with any UEFI compatible bios of the same architecture. We can imagine giving the UKI to uboot or ipxe to boot it in even greater number of environments.

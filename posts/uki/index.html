Unified Kernel Image

A Unified Kernel Image (UKI) is a combination of an UEFI boot stub program, a Linux kernel image, an initrd, and further resources in a single UEFI PE file. This file can either be directly invoked by the UEFI firmware or through a boot loader.

UKIs can run on UEFI systems and simplifies the distribution of small kernel images, for example it simplify network booting with IPxe.

UKIs make rootfs and kernels composable, making possible to derive a rootfs to multiple kernel version with one file for each pairs.

The simple way

If you use a systemd based OS and you want to use UKIs, you're probably better off using systemd-boot. In this post with want to create a ready to boot UKI for another machine than the one we creating it from.

Previously we build a minimal rootfs that we will be able to use in our UKI

Linux EFI

Linux kernel should be compiled with CONFIG_EFI_STUB=y https://docs.kernel.org/admin-guide/efi-stub.html

With this you can already boot linux directly from the UEFI

SD Boot

UEFI doesn't support UKIs directly hence we need glue code to copy UKI sections to memory and pass control to Linux.

We need to compile systemd-boot as the bootloader, to put in the UKI so it copies linux and initrd at the right place, display the splash screen measure secureboot values and finally give control to linux with the cmdline and dtb.

Create UKI

Creating an UKI involves mainly objcopy with the right parameter to pack all data into one PE file

we have sd_boot.efi our base UKI image, we now just have to fill the aforementioned sections.

the compisition can be done in any order using multiple call to objcopy

sd_boot randomize the virtual memory address (vma) so we first need to get the IMAGE_BASE to put the section after the sd_boot.

we give enough room between sections vma so the don't overlap for big files

IMAGE_BASE=$(objdump -x "sd_boot.efi" | grep ImageBase | awk '{print $2}')
objcopy \
    --add-section .osrel=osrel.txt \
    --add-section .cmdline=cmdline.txt \
    --add-section .sbat=sbat.txt \
    --add-section .initrd=initramfs.cpio \
    --add-section .linux=vmlinux \
    --change-section-vma .osrel=$( (( off = 16#00020000 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --change-section-vma .cmdline=$( (( off = 16#00020200 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --change-section-vma .sbat=$( (( off = 16#00020A00 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --change-section-vma .initrd=$( (( off = 16#00100000 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --change-section-vma .linux=$( (( off = 16#02000000 + 16#$IMAGE_BASE )); printf 0x%x $off ) \
    --set-section-alignment .sbat=512  \
    --set-section-flags .linux=code,readonly \
    sd_boot.efi uki.efi

Qemu Ipv6 Slirp
Posted on 3024-02-22 DRAFT

Qemu support rootless routing of network packets through the host using Slirp, you only need to give the right options to qemu to set this up, no fiddling with iptable or tap interface. But if you want to use it with ipv6 it might be a little tricky,

the documentation has no Ipv6 examples.

One would guess that using a Unique local address would work, but Qemu will only route packet out of the guest for Ips in the specified network.

The trick is to give ::/0 for the network, this way Qemu will route everything out of the guest.

This will allocate an Ipv6 on your router network

Option

Hence the options are:

-device virtio-net-pci,netdev=n0,mac=52:54:12:34:56:00 \
-netdev user,id=n0,hostfwd=tcp::5022-:22,ipv4=off,ipv6=on,ipv6-net=::/0

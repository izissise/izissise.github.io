Basic auth cookie

We want to limit access to certain files, to do so, let's implement authentification.

Because browsers have no way to set a custom http headers for links in <a> tags, we are force to use cookies, browsers will send cookies for every requests once it is setup for a url's domain.

We add a new login location and use nginx basic auth to check login/password pair to set a cookie.

...
server {
    ...
    location @login_success {
        add_header Set-Cookie "ngxp=???; max-age=15552000; path=/; SameSite=Lax; Secure; HttpOnly";
        return 200 "login success";
    }
    location ^~ /___ngxp/login {
        auth_basic "Login";
        auth_delay 1s;
        auth_basic_user_file /opt/ngxp/basic.htpasswd;
        try_files _ @login_success;
    }
}

we also need to create basic.htpasswd file that will contain credentials, we can use openssl cli:

printf '%s:%s\n' "$user" "$(openssl passwd -5 "$password")" >> basic.htpasswd

It works we can get a cookie asssigned with the login endpoint, but what do we put in the cookie?

We need to keep in the cookie:

  • username
  • a secret, so the cookie cannot be hand made to get access
  • the path base the user has access to

We can use nginx map module to define our entries

map $username $user_cookie { # map input output
    default "";
    admin   admin:d31d6b17bbc4df3ccb78d99db085e9d347b3e042e65d5878:/;
    alice   alice:2406b10bfd4c356394a647d5657bde937124f602198f20f3:/alicefiles;
}

Now we can set the cookie on login_success

    location @login_success {
        set $username $remote_user; # $remote_user is set with the basic_auth user in Authorization http header
        add_header Set-Cookie "ngxp=$user_cookie; max-age=15552000; path=/; SameSite=Lax; Secure; HttpOnly";
        return 200 "login success";
    }

The only thing left to do is to check our cookie is valid and can access the current uri when listing files. $cookie_ngxp contains the client provided cookie, it is an unstrusted user input, so we should check that it is exactly the same string has the one set for the username.

# get username from client sent cookie
map $cookie_ngxp $username {
    ~^(?<var>[^:]+):.* $var;
    default            "";
}
map ${cookie_ngxp}?${user_cookie} $user_cookie_equal {
    ~^([^\?]+)\?\1$   1; # if $cookie_ngxp == $user_cookie
    default           0;
}

Modifying location / a bit, we can already restrict listing and download for requests with a valid cookie:

...
server {
    ...
    location / {
        if ($user_cookie_equal = 0) { return 401; }
        ...
    }
}

We also want to have access right per path, using map again, can match that $uri starts with the extracted path in $user_cookie:

...

map $user_cookie $user_cookie_access {
    ~^([^:]+):([^:]+):(?<var>[^:]+)$ $var;
    default                          "";
}
map ${user_cookie_access}?${uri} $uri_access { # request uri access
    ~^([^\?]+)\?\1.*$ 1; # if $uri startwith $user_cookie_access
    default           0;
}
map ${user_cookie_equal}${uri_access} $user_authorized {
    ~^11$       1; # everything good authorize
    default     0;
}

server {
    ...
    location / {
        if ($user_authorized = 0) { return 401; }
        ...
    }
}

We now have per user/path access!

upload chunk from js and reconstruct on server

Now lets see how we can have files upload by users, this is a bit hacky but it works.

Since nginx does not really support file uploads, we will have to write most of the logic in javascript.

First we trick nginx to store the file on disk by setting up an upstream:

server {
    listen [::1]:4000;
    location /POST { return 204; }
    location /GET  { return 200; }
}

/POST must return 204 when the upload went well. Then we setup an endpoint that will actually store the file on disk:

...
server{
    ...
    location ^~ /___ngxp/upload/ {
        limit_except GET POST     { deny all; }
        if ($user_authorized = 0) { return 401; } # auth works here aswell

        client_body_temp_path      /home/user/uploads/; # upload path
        client_body_in_file_only   on; # store on disk
        client_body_buffer_size    16m;
        client_max_body_size       256m;
        client_body_timeout        2h;
        proxy_set_body             off; # do not send file to proxy
        proxy_pass                 http://[::1]:4000/$request_method;
    }
}

With client_body_in_file_only nginx will store the request body in a file at the path specified with client_body_temp_path, this works only if a proxy_pass is setup, with proxy_set_body off; the body will not actually be sent to the proxy.

Nginx will create a numbered files for every request bodies, but we would like to have the specified filename and also be able to upload files larger than client_max_body_size.

Using some javascript, we can upload a file with an XHR request but we must first split up the file in chunks, conveniently browsers File objects have a split function.

We also want to send a meta file for the server operator to be able to reconstruct the file from the chunks, we need a magic header, chunk size, chunk count, chunk last size.

// files is [File]
Array.from(files).forEach((f) => {
    var chunk_cnt = 1;
    var chunk_size = f.size;
    var chunk_last_size = 0;
    if (upload_max_size > 0 && f.size > upload_max_size) {
        chunk_cnt = Math.ceil(f.size / upload_max_size);
        chunk_size = Math.floor(f.size / chunk_cnt);
        if ((chunk_cnt * chunk_size) < f.size) {
            chunk_last_size = f.size - (chunk_cnt * chunk_size);
            chunk_cnt += 1;
        }
    }
    var meta = "#ngxpupload_meta\n" + JSON.stringify({
        'name': f.name,
        'size': f.size,
        'type': f.type,
        'chunk_cnt': chunk_cnt,
        'chunk_size': chunk_size,
        'chunk_last_size': chunk_last_size,
    });
    upload(upload_endpoint, meta).then((_) => {
        // meta upload success, start real upload
        var promise_chain = Promise.resolve();
        var seeker = 0;
        for (var i = 0; i < chunk_cnt; i++) {
            var chsz = chunk_size;
            if ((seeker + chsz) > f.size) {
                chsz = chunk_last_size;
            }
            let chunk = f.slice(seeker, seeker + chsz); // <--- slice
            seeker += chsz;
            promise_chain = promise_chain.then(() => upload(upload_endpoint, chunk));
        }
        promise_chain.then(upload_success);
    }, upload_error);
});

With the promise_chain each chunk of our file will be uploaded one after the other!

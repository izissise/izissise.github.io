Please
Posted on 3024-02-21 DRAFT

As I understand please has 2 steps that works in //,

the parse step, all BUILD file in the FS tree will be parsed, and if a subinclude is encounter in file, the target passed as parameter to subinclude is schedule for build and the parser wait until it is build to continue parsing the file the build step, all declared build_rules found at the parse step have a corresponding name and you can build a specific target with please build //path:target, the target dependencies (srcs, deps, ...) and recursively are scheduled for build. If the deps hashes of a target match the cache, just take the cache result, if not rebuild the dep.

Genesis

After watching LiveOverflow video about CTFs and him creating his own (http://maze.liveoverflow.com/#rules), I want to try to solve it using his mitm proxy that he describe in this (video)[https://www.youtube.com/watch?v=iApNzWZG-10].

But me already doing python in my daily job, and I wonder how it is to do something similar with something else than python.

Some constraints

The proxy tunnel part should be a standalone program that can act a lots like socat, it's purpose it to always keeps the tunnel open hence allowing the 'parsing' program to reload.

Wait maybe all I need is socat and some bash pipe magic

More than 4 hours later

After lots of stumbling with bash and named pipe, it seems it works!

Bash and named pipe is really error prone, bash will not launch a process that redirect to/from a named pipe if the other end isn't also open (but bash -x will display the line anyway) leading to great confusion.

In the end it mostly works, but pipes being stream based and not message based it means that we loose the information about the packets separation and can make the applications behave differently.

Even more time later

Unix fifos are not a good idea, in most case it is much more reliable to spawn to socat on a predefined port to make to process communicate.

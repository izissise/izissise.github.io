Ssh multi hostname
Posted on 3025-10-19 DRAFT 6 minutes read

TLDR: RaceConnectSocket

Introduction

Recently I wanted to connect to a local machine with ssh but didn't know it's IP, I known the MAC address which I could translate into the IPv6 link-local address but since the connect syscall requires the local iface name for ip6 link local, it would need to be hardcoded in the ssh config, making it inconvenient to change. Also if the same machine is accessible from a wider network, I'd need a second entry in the config and have to remember which one to use depending on which network my computer is connected to.

That’s a lot of complexity and I thought most of it could be solved if ssh config allowed multiple hostnames for a single host entry.

Sadly OpenSSH's Hostname doesn't support multiple entries:

Host machineA
  Hostname machinea.local 192.168.1.65 2a00:1450:4007:81a::200e 142.250.201.174 00:00:07:00:00:00 ...
  Port 22
  User admin
  PubkeyAuthentication yes
  IdentityFile ~/.ssh/id_rsa

Actually, there’s a silver lining, ProxyCommand. It runs a program that’s expected to connect to the remote machine, OpenSSH will simply speaks its protocol through the program's stdin and stdout!

Which means I can do the following:

Host machineA
  ProxyCommand program %p machinea.local 192.168.1.65 2a00:1450:4007:81a::200e 142.250.201.174 00:00:07:00:00:00 ...
  Port 22
  User admin
  PubkeyAuthentication yes
  IdentityFile ~/.ssh/id_rsa

I just need to figure out what program is.

Socat

It feels like socat with a wrapper shell script would be perfect for the job, I successfully got a serialized version to work, but since it tried every hostname one after another, it could take a long time to connect if only the last one on the command line was reachable.

The main issue I had was that socat executes the script defined on the exec statement before it has successfully connected, making it hard to know which socat instance is connected when spawning multiple ones in parallel..

stackoverflow

In the end, I decided to write this program myself.

RaceConnectSocket

RaceConnectSocket or rcs which takes hostnames and ips has arguments and will spawns threads to concurrently connect to all of them, keeping the first successful connection and then pipe bytes through stdin/stdout.

Git repository

It is written in C using cosmopolitan libc, Some reasons for this choice:

  • Run on all major oses with one binary
  • Static binary
  • Easy 'C' development thanks to sane api defaults
  • Debug enviroment included

Also I'm addicted to sugar, I also added pretty.c in the mix.

Steps

I want to connect to the ssh port for each hostname on the command line in a parallel fashion. On the first successful connection, pipe through stdin/stdout to the socket and closes other connections.

The first step is to resolve every hostname to a list of IPv4 or IPv6 to give to the connect syscall.

Ipv6 local link

Also some notes on IPv6 link local addresses,

IpV6 has a set of special addresses called link local in the range fe80::/10. All network interface must be assigned one if it wants to use Ipv6. These addresses are not routed by routers and allow for ipv6 devices to talk to each other without a router on the same link.

Because ip6 link local addresses are usually derived from the network interface's mac address, that makes them predictable and so I would be nice to be able to use them to connect to the remote machine.

The problem is that, unlike IPv4 and global IPv6 addresses (where only the destination address is needed), link-local addresses require specifying the interface index in struct sockaddr_in6 when using connect.

Since I don’t want to ask the user which network interface shares a link with the remote machine, I simply perform a connect syscall for every network interface on the local machine.

So a list of locally attached network interfaces is needed, but as of 2025 getifaddrs in cosmopolitan is only implemented for IPv4.

That became a whole side quest, implemented across 3 PRs:

  • Linux
  • BSD
  • Windows

DNS resolution

In cosmopolitan, hostname lookup with getaddrinfo is the courtesy of the musl libc implementation. As of 2025, it doesn't support mDns :/

getaddrinfo returns a linked list of struct addrinfo which are the ips 4 and 6 the name resolve to.

If it returns an Ipv6 link-local address we have to enter the special case described earlier.

Concurrent connections

Because getaddrinfo getaddrlist connect etc are blocking calls, I spawn one thread per arguments on the command line with a small stack size (see cosmo's greenbean).

This way all names are tried simultaneously, making the connections truly racy.

IO fun

To ensure only one connection is kept, I use an atomic variable that will only be set once.

The winning connection will spawns two final threads that handle reading and writing byte to/from the socket and standard input/output.

pthread timeout

Because I want rcs to fail fast when no hostname is reachable and there is no easy way to specify to connect or getaddrinfo a timeout, I spawn a sibling thread before calling connect and getaddrinfo that will pthread_cancel on the original thread after a timeout.

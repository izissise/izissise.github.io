Nginx Explorer - File listing
Posted on 3024-10-20 DRAFT

TLDR

Nginx explorer is a minimal web interface to download/upload files.

git clone https://github.com/izissise/nginx-explorer.git
nginx-explorer/ngxp.sh servethis

Nginx configuration

Genesis

It is always a struggle to share files with other computers or phones.

Everything talk HTTP these days and HTTP support file download and upload!

The easy way is to just use python -m SimpleHTTPServer, but the performance for big files aren't great.

There is also miniserve which is great with a lot of features.

But let's use nginx, it has a builtin recursive file listing options and it is a battle tested webserver with great performances!

How does it work?

In multiple steps,

we configure nginx to list files using autoindex module:

charset UTF-8;
server_tokens off;

server {
    listen 8080;
    location / {
        root /home/user/downloads;
        autoindex on;
        autoindex_format html;
    }
}

Now let's add some personalization, nginx autoindex start the response with <html> tag, we can use nginx sub_filter to replace the tag with our custom html:

...
map $uri $cond_filter {
    ~/$ "<!DOCTYPE html><html><link rel='stylesheet' href='/___ngxp/main.css' /><script src='/___ngxp/main.js'></script>";
    default "<html>";
}

server {
    ...
    location / {
        ...
        sub_filter_once on;
        sub_filter "<html>" $cond_filter;
    }
}

There are multiple things happening there:

  1. we want to replace the tag only once sub_filter_once on;
  2. we don't want files download to be modified, using map $uri we inject html only for directories (when the uri ends with a '/')
  3. for some reasons autoindex does not output <!DOCTYPE html>, so we add it.

Now we successfully injected html on our file listing, but the main.css and main.js are not files that we want to appear in the listing. We can add a new location that point to another filesystem path than the one that we are listing:

...
server {
    ...
    location ^~ /___ngxp/ {
        alias /var/www/ngxp/;
    }
}

We use alias because we don't want to keep the /___ngxp/ part to find files.

So going back to our docker container, we have two directory to mount + nginx configuration files:

docker run \
        --rm -it --log-driver=none \
        --user="$(id -u):$(id -g)" \
        --userns=keep-id --cap-drop=ALL \
        --tmpfs=/tmp:rw,noexec,nosuid,size=70m \
        --expose=8080 -p 8080:8080 \
        -v "/tmp/nginx.conf:/etc/nginx/nginx.conf:ro" \
        -v "/tmp/nginx-explorer.conf:/etc/nginx/conf.d/default.conf:ro" \
        -v "/tmp/main.css:/var/www/ngxp/main.css:ro" \
        -v "/tmp/main.js:/var/www/ngxp/main.js:ro" \
        -v "$HOME:/home/user/downloads:ro" \
        nginx

nginx.conf is a special nginx configuration so nginx works with our read only container[^1]. nginx-explorer.conf is the configuration we just built. $HOME is the path that will be listed.

Nginx Explorer - Upload
Posted on 3024-11-23 DRAFT 2 minutes read

Now lets see how we can have files upload by users, this is a bit hacky but it works.

Since nginx does not really support file uploads, we will have to write most of the logic in javascript.

First we trick nginx to store the file on disk by setting up an upstream:

server {
    listen [::1]:4000;
    location /POST { return 204; }
    location /GET  { return 200; }
}

/POST must return 204 when the upload went well. Then we setup an endpoint that will actually store the file on disk:

...
server{
    ...
    location ^~ /___ngxp/upload/ {
        limit_except GET POST     { deny all; }
        if ($user_authorized = 0) { return 401; } # auth works here aswell

        client_body_temp_path      /home/user/uploads/; # upload path
        client_body_in_file_only   on; # store on disk
        client_body_buffer_size    16m;
        client_max_body_size       256m;
        client_body_timeout        2h;
        proxy_set_body             off; # do not send file to proxy
        proxy_pass                 http://[::1]:4000/$request_method;
    }
}

With client_body_in_file_only nginx will store the request body in a file at the path specified with client_body_temp_path, this works only if a proxy_pass is setup, with proxy_set_body off; the body will not actually be sent to the proxy.

Nginx will create a numbered files for every request bodies, but we would like to have the specified filename and also be able to upload files larger than client_max_body_size.

Using some javascript, we can upload a file with an XHR request but we must first split up the file in chunks, conveniently browsers File objects have a split function.

We also want to send a meta file for the server operator to be able to reconstruct the file from the chunks, we need a magic header, chunk size, chunk count, chunk last size.

// files is [File]
Array.from(files).forEach((f) => {
    var chunk_cnt = 1;
    var chunk_size = f.size;
    var chunk_last_size = 0;
    if (upload_max_size > 0 && f.size > upload_max_size) {
        chunk_cnt = Math.ceil(f.size / upload_max_size);
        chunk_size = Math.floor(f.size / chunk_cnt);
        if ((chunk_cnt * chunk_size) < f.size) {
            chunk_last_size = f.size - (chunk_cnt * chunk_size);
            chunk_cnt += 1;
        }
    }
    var meta = "#ngxpupload_meta\n" + JSON.stringify({
        'name': f.name,
        'size': f.size,
        'type': f.type,
        'chunk_cnt': chunk_cnt,
        'chunk_size': chunk_size,
        'chunk_last_size': chunk_last_size,
    });
    upload(upload_endpoint, meta).then((_) => {
        // meta upload success, start real upload
        var promise_chain = Promise.resolve();
        var seeker = 0;
        for (var i = 0; i < chunk_cnt; i++) {
            var chsz = chunk_size;
            if ((seeker + chsz) > f.size) {
                chsz = chunk_last_size;
            }
            let chunk = f.slice(seeker, seeker + chsz); // <--- slice
            seeker += chsz;
            promise_chain = promise_chain.then(() => upload(upload_endpoint, chunk));
        }
        promise_chain.then(upload_success);
    }, upload_error);
});

With the promise_chain each chunk of our file will be uploaded one after the other!
